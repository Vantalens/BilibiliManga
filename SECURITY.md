# BiliManga SECURITY

版本：v0.1.0
作者：Codex
状态：草稿
最后更新：2026-07-03

## 安全边界

- 本项目只处理用户已授权、已购买或公开可访问内容。
- 支付、充值、购买、解锁、权益变更由官方网页处理。
- 本地数据库全库加密，密钥使用系统密钥环或等价安全存储。
- 日志不得包含 Cookie、Token、账号标识、订单信息或图片真实鉴权 URL。当前登录 gate 不保存 Cookie、Token 或账号标识。

## 接口调研安全要求

- 只记录公开页面、公开静态资源或用户授权路径中观察到的接口。
- 响应样例必须脱敏；不得保存 Cookie、Token、账号、订单、密钥或图片真实鉴权 URL。
- 钱包、券、余额、订单、支付、充值、购买、解锁和权益变更端点只能标记为禁止原生实现。
- 允许类接口必须先有 schema 测试、失败模式和降级路径，再接入 UI 或 Rust bridge。
- 当前 Rust `search_suggestions` 只接入 verified 的公开 `SearchSug`，不发送 Cookie，不处理账号态，不调用关键词 `Search`、详情、图片或权益端点。

## 缓存策略

- 图片缓存只用于短期性能优化。
- 缓存不可导出，不提供永久离线包。
- 必须支持容量上限、过期清理和一键清除。
- 缓存索引必须存入 SQLCipher，缓存文件删除必须拒绝绝对路径和父级路径跳转。


## 当前实现状态

- Rust 后端已提供 `initialize_secure_storage`，会通过系统密钥环生成或读取数据库密钥，不向前端返回密钥。
- Rust 后端已提供 `clear_image_cache`，只清理应用缓存目录下的图片缓存，不接受前端传入任意路径，并同步清空 SQLCipher 缓存索引。
- Rust 后端已提供 `image_cache_status` 和 `prune_image_cache`，按缓存索引执行过期优先、LRU 次之的容量清理，且缓存相对路径会做目录逃逸检查。
- SQLCipher 实际数据库文件验证已完成；真实书库、进度和阅读偏好写入必须使用该加密数据库路径。书库管理字段包括分组、标签、评分、备注和未读章节。
