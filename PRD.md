## 背景

网页版哔哩哔哩漫画在桌面长期阅读和本地书库管理上不够舒适。BiliManga 的第一版目标是提供 Windows 桌面阅读闭环，并为后续 macOS/Linux 支持保留架构空间。

## 目标

- P0：登录后从书架或搜索进入漫画，连续阅读 1-2 小时不中断。
- P0：支持浏览未购买漫画的公开信息、章节目录和免费/试读章节；锁定章节只走官方网页。
- P0：保存阅读进度、阅读器偏好、本地标签、分组、评分、备注和统计。
- P0：短期图片缓存提升阅读体验，但不支持导出、永久离线包或批量下载。
- P1：提供 Windows 安装包和签名自动更新能力。

## 非目标

- 不实现支付、充值、购买、解锁、权益变更接口。
- 不实现未解锁章节图片获取、批量下载、导出、第三方共享 Cookie 托管或绕过权益的解析链路。
- 不绕过付费、鉴权、风控或官方访问限制。
- 不在 MVP 覆盖网页版首页、分类、排行、专题的完整功能。
- 不做多设备同步、批量编辑、导入导出。

## 验收标准

- Given 用户已登录且账号有可访问漫画，When 从书架打开漫画并阅读 1-2 小时，Then 阅读器不崩溃且进度可恢复。
- Given 用户切换长滚动和分页模式，When 退出并重启应用，Then 模式和页码按本地偏好恢复。
- Given 用户打开未购买漫画，When 详情接口返回公开信息和章节列表，Then 应用展示详情、章节目录和免费/试读入口，但锁定章节不渲染图片。
- Given 章节未解锁，When 用户点击阅读，Then 应用打开官方网页处理，返回后可以刷新权益状态。
- Given 缓存超过容量或过期，When 清理任务运行，Then 只删除短期缓存，不删除书库和进度数据。
